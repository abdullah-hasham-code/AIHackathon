Hackathon Task Plan

Team setup:
- 3 Developers
- 1 Presenter

Tasks and ownership:
1) Problem analysis - All developers
2) Problem selection - All developers + Presenter
3) Business solution brainstorming - All team members
4) UI design on paper - Dev 1 + Dev 2
5) UI development - Dev 1 + Dev 2 + Dev 3
6) Testing - Dev 3 (lead) + Dev 1
7) Demo video script and recording - Presenter (lead) + Dev 2
8) Final submission (6 PM to 8 PM PST) - Presenter + Dev 1

Execution schedule (PST):
10:00 AM - 11:00 AM
- Problem analysis and idea shortlisting (all team members)

11:00 AM - 12:00 PM
- Final problem selection and scope lock
- Define success criteria and must-have features

12:00 PM - 1:00 PM
- Business solution brainstorming
- Freeze MVP feature list

1:00 PM - 2:00 PM
- UI paper sketches and user flow finalization (Dev 1 + Dev 2 + Presenter)

2:00 PM - 4:00 PM
- Core UI and app development (Dev 1 + Dev 2 + Dev 3)
- Presenter prepares demo storyline and value proposition

4:00 PM - 5:00 PM
- Testing, bug fixing, and UX polish (Dev 3 lead, Dev 1 support)
- Presenter drafts demo script

5:00 PM - 6:00 PM
- Demo video recording and final edit (Presenter lead, Dev 2 support)
- Final submission checklist preparation (Dev 1)

6:00 PM - 7:00 PM
- Final QA pass and packaging
- Submit early if ready

7:00 PM - 8:00 PM
- Buffer for upload issues, last-minute fixes, and final submission confirmation
